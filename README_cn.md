# 控制器：从 PID 到 QP-MPC

交互式仿真器，追溯反馈控制的演化——从经典 PID 到 LQR 再到带约束 QP-MPC——全部应用在同一台直流电机伺服系统上。

**核心理念：** 三代工程师看着同一台电机，给出了三种截然不同的答案。不是因为电机变了，而是因为*他们手里有什么东西*变了——他们懂的数学、他们能摸到的计算机、他们被雇来解决的问题。

## 你将学到什么

**PID 探索器**——控制器增益如何重塑闭环动态：
- Kp 提高有效 ωₙ（响应更快），但降低有效 ζ（超调更大）
- Kd 增加阻尼——将主导极点向左拉，提升 ζ
- Ki 通过在 s=0 处引入积分器极点来消除稳态误差
- PD 让你可以独立调节 ζ_eff 和 ωₙ_eff；PI 消除静差但会拖长调节时间

**伺服电机 PID**——把 PID 应用到真实硬件上会发生什么：
- 电压饱和会削平你的指令——抗积分饱和让积分器保持理智
- 转矩扰动造成只有 Ki 能消除的稳态误差
- 刚性电气动态（小 L 值）需要降阶模型才能稳定仿真

**LQR 探索器**——最优控制与经典调参有何不同：
- 你调的"不是"增益——你声明你在意什么（Q、R 权重），数学返回最优 K
- LQR 保证 PID 无法承诺的稳定裕度（≥60° PM，∞ 幅值裕度）
- LQI 通过状态增广加入积分作用——零稳态误差，有保证
- 参考前馈不可或缺：没有它，LQI 需要约 7 秒才能到达设定值（有前馈只需 0.5 秒）

**QP-MPC 探索器**——当约束不可谈判时会发生什么：
- LQR 会命令最优性要求的任何电压——QP-MPC 找到 12V 电源*实际能输出*的最好电压
- 滚动时域优化：每个时间步求解一个带约束 QP，只施加第一步，然后重复
- 约束不活跃时，QP-MPC *就是* LQR——平滑、自动的模式切换
- 预测时域可视化展示控制器计划要走的路径

**零点效应探索器**——零点如何改变二阶系统的阶跃响应：
- LHP 零点：速度-超调权衡——这就是微分作用在做的事
- RHP 零点：硬性物理限制——系统先朝反方向运动，任何控制器都无法消除
- 多个零点效果叠加——每增加一个微分项就多一次冲击

### 🔑 一个核心洞见：积分器极点会移动

当你滑动 Ki 滑块时，会立刻看到以下现象——这**不是**计算错误：

PID 控制器的积分项（Ki/s）在 **s = 0** 处引入一个极点。在**开环**中，这个极点就停在原点。但当反馈回路闭合后，它**移动了**。以"PID Tuned"预设为例（ζ=0.5, ωₙ=2, Kp=3, Kd=0.8）：

| Ki | 实极点（原本在 s=0） | 共轭复极点对 | 现象 |
|----|----------------------|-------------|------|
| 0（PD） | 抵消 | −2.6 ± j3.0 | 无积分器——稳态误差残留 |
| 0.4 | **−0.103** | −2.55 ± j3.0 | 实极点靠近原点 → 拖出长长的调节尾巴 |
| 2 | −0.48 | −2.36 ± j3.2 | 积分器加速，共轭复极点对向右漂移 |
| 10 | −3.77 | −0.71 ± j3.2 | 共轭复极点对逼近虚轴——Ki 太大了！ |

**这是 PID 控制的基本特性**：积分作用将稳态误差拉向零，但随着 Ki 增大，实极点向左移动（积分器加快），而共轭复极点对向虚轴靠近。Ki 调得太大，复极点将越过虚轴进入右半平面——系统失稳。

从根轨迹的角度看：积分器极点从 s=0 出发沿负实轴移动，而两个对象极点先彼此靠拢，然后随 Ki 增大分离。PID 探索器实时展示这一切——滑动 Ki 时请紧盯零极点图。

### 为什么闭环极点数量随 Ki 变化

一个微妙但重要的细节：切换到 PD 预设（Ki=0），你会看到 **3** 个闭环极点。切换到 PID（Ki>0），你会看到 **4** 个。这不是 bug——是数学的直接结果。

电机的开环传递函数是三阶的：

$$G(s) = \frac{K_t}{LJ s^3 + (LB+RJ) s^2 + (K_t^2+RB)s}$$

分母中的 $s$ 因子是机械积分器（角度 = ∫ 角速度 · dt）。PID 控制器 $C(s) = K_p + K_d s + K_i/s$ 通过 $K_i/s$ 项再增加一个积分。

- **Ki > 0（完整 PID）：** 为了消除 $K_i/s$ 项，必须两边同乘 $s$，得到 **四阶四次方程**：$LJ s^4 + (LB+RJ)s^3 + (K_t^2+RB+K_t K_d)s^2 + K_t K_p s + K_t K_i = 0$。四个极点。
- **Ki = 0（PD）：** 没有 $K_i/s$ 需要消除。特征方程直接是 **三阶三次方程**：$LJ s^3 + (LB+RJ)s^2 + (K_t^2+RB+K_t K_d)s + K_t K_p = 0$。电机在 $s=0$ 处的开环极点在 PD 反馈下**移动了**——$K_p$ 和 $K_d$ 将其沿负实轴向左推动。三个极点。

在 $K_i=0$ 时额外乘以 $s$ 会在原点引入一个虚假根——这是数学操作带来的伪影，而非物理极点。代码正确处理：PD 时使用三次方程求解器，完整 PID 时使用四次方程求解器。

## LQR 探索器

`lqr_explorer.html` 在同一有刷直流电机上展示了 **LQR（线性二次型调节器）** 设计——PID 的现代状态空间对应方法：

- **LQI（LQR + 积分作用）**——将三阶电机模型扩展一个误差积分状态，实现零稳态跟踪误差
- **调节权重，而非增益**——不再试探性地调整 Kp/Kd/Ki，而是指定**代价权重** Q（状态惩罚）和 R（控制量惩罚）；最优状态反馈增益 K 自动计算得出
- **CARE 求解器**——通过 Hamiltonian 方法求解连续时间代数 Riccati 方程，使用 `numeric.eig()` 对 8×8 Hamiltonian 矩阵进行特征值分解
- **实时阶跃响应**——双 Y 轴图：左轴 θ [rad]、右轴电压 V [V]；接近 ±Vmax 时显示饱和填充区域
- **实时零极点图**——4 个 LQR 闭环极点（青色 ×）对比 3 个开环电机极点（灰色 ×）；高亮稳定性边界
- **对数映射的代价权重**——滑块对应 q_θ（位置误差）、q_ω（速度）、q_i（电流）、q_∫（积分误差）和 R（控制量惩罚）
- **PID 对比开关**——叠加 PID 阶跃响应（金色虚线），直观对比经典整定与最优 LQR
- **增益读数**——K = [k_θ, k_ω, k_i, k_∫]，颜色编码对应 PID 概念：k_θ≈Kp、k_ω≈Kd、k_i（电流反馈）、k_∫≈Ki
- **6 个预设**——均衡 LQR、低成本控制、高成本控制、位置优先、重阻尼、快速积分器
- **抗积分饱和**——电压饱和时冻结积分器状态，防止积分器卷绕
- **自适应时间窗口**——仿真时长根据闭环调节时间自动缩放（4/(ζ·ωₙ)）

### LQR vs PID：核心区别

| 方面 | PID | LQR |
|------|-----|-----|
| 整定方式 | 试探——Kp、Kd、Ki | 系统化——Q、R 权重 |
| 增益数量 | 3 个标量增益 | 4 个增益：k_θ、k_ω、k_i、k_∫ |
| 电流反馈 | 无（隐含） | 显式 k_i 反馈 |
| 设计方法 | 边试边看 | 一步最优 |
| 理论保证 | 无（可能失稳） | 保证稳定裕度（≥60° PM, ≥6 dB GM） |
| 工作量 | 直接调节响应形状 | 惩罚状态 + 控制量 → 最优折中 |

### LQI 代价函数

$$J = \int_0^\infty \left( q_\theta \theta^2 + q_\omega \omega^2 + q_i i^2 + q_{\int} (\smallint e)^2 + R u^2 \right) dt$$

- **q_θ ↑**——位置响应更快，超调增大
- **q_ω ↑**——阻尼增加，超调减小
- **q_i ↑**——电流消耗更小
- **q_∫ ↑**——稳态误差恢复更快
- **R ↑**——控制量更小、更保守（惩罚电压）

CARE 求解器得出最小化 J 的最优增益矩阵 **K**。闭环特征值 (A − BK) 即为 LQR 最优极点位置——无需手动配置极点。

### 🔑 LQR 如何处理参考指令

每个 LQR 新手都会问的问题：**参考输入 `r` 在哪里？** 控制律 `u = −Kx` 只涉及状态——它把所有状态驱动到零。那 LQR 如何跟踪期望位置？

**LQR 是调节器，不是跟踪器。** 它被设计用来将受扰系统恢复到平衡点（`x → 0`），而非跟踪指令。要让它跟踪，必须将跟踪问题转化为调节问题。有两种标准方法。

**方法一：LQI（LQR + 积分作用）。** 用跟踪误差的积分来增广状态：

$$x_{aug} = [\theta, \omega, i, \smallint e]^T, \quad e = r - \theta$$

增广动力学通过误差积分注入参考指令：

$$\dot{x}_{aug} = A_{aug} x_{aug} + B_{aug} u + \underbrace{[0, 0, 0, 1]^T}_{B_{ref}} \cdot r$$

参考 `r` 通过驱动 $\smallint e$ 的变化率进入系统：$d(\smallint e)/dt = e = r - \theta$。最优控制 `u = −K·x_aug` 自动包含积分作用，因为 $x_{aug}$ 包含 $\smallint e$。这保证了对恒定参考的**零稳态误差**——积分器会一直调整直到 $\theta = r$。

**方法二：参考前馈。** 直接在控制律中加入前馈项：

$$u = N r - K x$$

`N` 的选择使从 `r` 到输出的直流增益为 1。没有 N 时，闭环系统的直流增益通常不为 1，即使没有扰动，$\theta_{ss} \neq r$。

**为什么纯 LQI 感觉慢——以及 `lqr_explorer.html` 如何解决。** 在方法一中，参考只能*间接*通过误差积分器到达被控对象。$t=0$ 时所有状态为零，$u = 0$——没有即时控制动作。积分器必须随时间积累误差才能产生有意义的电压。对默认电机，这需要约 7 秒才能到达设定值。

解决方案：结合两种方法。用 $k_\theta r$ 作为前馈（类比 PID 的 $K_p r$），同时保留积分状态进行稳态修正：

$$u = k_\theta r - K x_{aug} = k_\theta(r-\theta) - k_\omega \omega - k_i i - k_{\smallint} \smallint e$$

这是 `lqr_explorer.html` 的默认行为。以下是各方法注入参考指令的方式对比：

| 方法 | 控制律 | $t$=0 时 $u$ | $\theta_{ss}$ | 速度 |
|------|--------|-------------|---------------|------|
| PID（PD, Ki=0） | $u = K_p(r-\theta) - K_d\omega$ | $K_p r$（5.7 V） | $r$ − 静差 | 快 |
| 纯 LQI（无前馈） | $u = -K x_{aug}$ | 0 V | $r$（通过 ∫） | 很慢 |
| LQI + $k_\theta$ 前馈 | $u = k_\theta r - K x_{aug}$ | $k_\theta r$ | $r$（通过 ∫） | 快 |
| LQI + 精确直流前馈 | $u = N r - K x_{aug}$ | $N r$ | $r$（通过 ∫） | 快，精确 |

$k_\theta$ 前馈是一个实用选择——它简单，正是 PID 所做的，任何残余直流偏移都由积分作用修正。精确的直流增益前馈 $N = -(C(A-BK)^{-1}B)^{-1}$ 可以完全消除偏移，但需要矩阵求逆且依赖于当前 K。

**简而言之：** LQR 需要帮助来跟踪参考。LQI 通过误差积分提供（保证零稳态误差，但慢）。加入 $k_\theta r$ 前馈提供即时"冲击"使其响应迅速。这不是取巧——这是 LQI 实现中的标准做法，直接类比 PID 的 $K_p r$ 项。

### 🔑 q_ω 速度悖论：快 → 慢 → 再快

在 LQR 探索器中试试：设 q_θ = 5，然后将 q_ω 从 0.01 滑到 100。你会看到阶跃响应**先快、后慢、然后再快**。这是正确的 LQR 行为，由**主导极点切换**引起——随着速度惩罚增大，主导极点发生了迁移。

以默认电机参数、q_θ = 5 为例：

| q_ω | k_θ | k_ω | 主导极点 | ζ_eff | 调节时间 | 现象 |
|-----|-----|-----|---------|-------|---------|------|
| 0.01 | 9.1 | 1.58 | 复共轭 −5.2 ± j5.0 | 0.72 | 0.78 s | 欠阻尼，**快速**上升但伴有振荡 |
| 0.1 | 9.4 | 1.86 | 复共轭 −6.2 ± j3.5 | 0.87 | 0.65 s | 阻尼增强，响应干净——最快稳定 |
| 0.3 | 9.9 | 2.36 | **实极点** −1.5（主导） | ~1.0 | 2.63 s | 极点切换！过阻尼，**变得迟钝** |
| 1.0 | 11.1 | 3.59 | 复共轭 −1.7 ± j0.6 | 0.95 | 2.37 s | 再次欠阻尼，但 ωₙ 较低 |
| 10 | 16.0 | 10.2 | 复共轭 −0.8 ± j0.6 | 0.79 | 5.06 s | 高 k_θ 带来强力冲击——**启动很快**，但尾巴很长 |

**为什么会这样：**

- **低 q_ω**——速度惩罚很轻。LQR 解几乎不加阻尼，主导极点是一对快速、轻阻尼的复共轭极点（ωₙ ≈ 7 rad/s）。响应快但有振荡。

- **中等 q_ω（~0.3）**——阻尼将复共轭极点推至最左端，分裂为两个实极点。此时积分作用极点（靠近 s = 0）成为**最慢**的极点。响应变为过阻尼，迟钝缓慢——积分器极点占据了主导地位。

- **高 q_ω**——两个实极点重新合并为一对慢速复共轭极点，而比例增益 k_θ 显著增大（从 9 增至 26）。高 k_θ 产生强烈的初始"冲击"，感觉很快，但主导极点很慢（ωₙ ≈ 0.6 rad/s），导致长长的调节尾巴。

滑动 q_ω 时紧盯零极点图——你会看到四个闭环极点经历这场切换，而控制器零点（橙色 ○）也讲述着同样的故事：一个零点始终靠近原点（对应慢尾巴），另一个则向左飞出很远（对应微分冲击）。

### 🔑 LQR 与 PID 对比：参考前馈

一个常见的第一印象：无论怎么调 Q/R 权重，LQR 总比整定好的 PD 控制器（例如 Kp=5.7, Kd=1.72, Ki=0）慢。这**不是** LQR 的基本局限——而是缺少前馈项。

**为什么 PID 看起来更快。** PID 控制律为：

$$u = K_p e - K_d \omega = K_p (r - \theta) - K_d \omega = \underbrace{K_p r}_{\text{前馈}} - K_p \theta - K_d \omega$$

在 $t=0$ 时，$K_p r$ 项立即施加 $K_p \cdot 1 = 5.7\mathrm{V}$ 到电机——一个瞬间"冲击"驱动响应。

**为什么 LQI 看起来更慢。** 纯 LQI 控制律为：

$$u = -(k_\theta \theta + k_\omega \omega + k_i i + k_{\smallint} \smallint e)$$

在 $t=0$ 时，所有状态为零，所以 $u=0$。控制器必须等待 $\smallint e = \int (r-\theta)dt$ 缓慢积累才能施加有意义的电压。默认配置下，到达设定值需要**约 7 秒**——而 PID 上升时间只需 0.54 秒。

**解决方案：参考前馈。** 在控制律中加入 $k_\theta r$（完全类比 PID 的 $K_p r$）：

$$u = k_\theta r - (k_\theta \theta + k_\omega \omega + k_i i + k_{\smallint} \smallint e) = k_\theta (r-\theta) - k_\omega \omega - k_i i - k_{\smallint} \smallint e$$

这是 LQR 探索器的默认行为。积分作用只需修正残余稳态误差——无需提供全部直流偏置。

**性能对比（默认电机，R=4, L=0.02, Kt=0.06, J=0.002, B=2e-4）：**

| 控制器 | 增益 | $t_r$ [s] | $t_s$ [s] | 超调 % | 稳态误差 % | 备注 |
|--------|------|-----------|-----------|--------|-----------|------|
| PD (Ki=0) | Kp=5.7, Kd=1.72 | 0.54 | 0.94 | 0 | 0.01 | 无积分——无法抑制扰动 |
| LQI (均衡, q_θ=100) | k=[33, 4.4, 0.7, −10] | 0.22 | 2.0 | 3.0 | 0.02 | 积分作用——保证零稳态误差 |
| LQI (快速, q_θ=500) | k=[71, 5.7, 0.9, −10] | 0.14 | 0.24 | 1.2 | 0.005 | 12V 饱和——激进 |
| LQI (小 q_∫, 低 R) | k=[22, 4.0, 0.6, −1.0] | 0.32 | 0.55 | 0.7 | 0.007 | 放松积分 ≈ 类 PD 动态 |

**如何调试 LQI 超越 PID 性能：**
- **q_θ ↑** — 增大 $k_\theta$（比例增益），提供更强的前馈冲击，加快上升。注意高值时的饱和。
- **q_ω ↑** — 增大 $k_\omega$（微分增益），增加阻尼。与 PID 不同，LQR 以最优方式完成——不会放大传感器噪声。
- **R ↓** — 减小控制量惩罚，允许更大增益（低成本控制）。这是 LQR 等效的"更激进整定"。
- **q_∫ ↓** — 缩小积分增益 $k_{\smallint}$，使响应更接近 PD。当扰动抑制不重要时使用。

## QP-MPC 探索器

`servo_qp_mpc.html` 演示了**二次规划模型预测控制**——伺服控制的第三代，约束是头等公民：

- **三阶电机模型**——位置 θ、速度 ω、电流 i——前向欧拉离散化，采样时间 Ts 可配置
- **在线 QP 求解器**——坐标下降法求解箱式约束二次规划：`min ½UᵀHU + fᵀU`，满足 `−Vmax ≤ uₖ ≤ +Vmax`
- **并排 LQR 仿真**——另一个电机实例在跑无约束 LQR，直观对比：QP-MPC 在 ±12V 处干净削顶，而 LQR 傻傻地命令 30V
- **预测时域可视化**——青色虚线点显示 MPC 在未来 N 步内的规划轨迹
- **可配置时域**——N = 3 到 30 步；时域越长看得越远，但计算量越大
- **实时 QP 诊断**——活跃约束数、QP 迭代次数、求解时间（ms）、代价函数值 J
- **三张实时图表**——位置跟踪 θ(t)、控制电压 V(t) 叠加 ±Vmax 限制线、电机电流 i(t) 叠加 ±Imax 限制线
- **电机转轴动画**——转子角度的可视化指示
- **自适应速度控制**——放慢仿真，逐帧观察约束边界行为

### QP-MPC vs LQR：约束的故事

| 场景 | LQR | QP-MPC |
|------|-----|--------|
| **约束不活跃** | 自由施加最优电压 | 与 LQR 一致——无约束最优解可行 |
| **电压撞到 +Vmax** | 饱和——最优性丢失，可能 windup | 找到约束最优解：Vmax 以内能做到的最好 |
| **从静止启动** | 命令大幅度脉冲以快速加速 | 同样的脉冲——但如果需要会削顶到 Vmax |
| **接近稳态** | 在小幅修正，完全在约束之内 | 与 LQR 一致——约束松弛 |

核心洞见：**约束不活跃时，QP-MPC *就是* LQR。** 没有切换、没有增益调度。QP 求解器在所有边界都松弛时自然恢复无约束最优解，在某条约束变得活跃时平滑过渡到沿边界行走的解。

### 🔑 最常见的误解："把 LQR 限幅不就行了"

这是一个陷阱。你有 LQR，它是最优的，你的驱动器只能输出 ±12V。显而易见的修补：照常计算 u = −Kx，然后一行代码限幅到 ±12V：

```
u = clamp(−Kx, −Vmax, +Vmax)
```

这被称为**朴素饱和（naive saturation）**，感觉很对。毕竟，在控制量"可及"的时候你在施加最优控制，不可及的时候你做了硬件允许的最好——对吧？

**不对。这是约束控制中唯一最重要的思想。**

对 LQR 输出限幅，得到的不是约束最优解。差得远。分两层来说：

**第一层——控制器不知道自己在饱和。** LQR 在计算增益时假设控制量无限。整个增益矩阵 K 是为无约束世界设计的。控制器规划的激进动作——大幅电压脉冲、骤然反向——只有在放大器真能实现时才有意义。事后限幅意味着控制器在盲目飞行：它持续计算指令，以为电机收到了 30V，而电机实际只收到 12V。内部状态轨迹偏离了控制器的认知。

一个具体例子：电机从静止启动，LQR 命令 30V 脉冲加速。你把它限到 12V。电机加速比 LQR 预期的慢。LQR 的状态估计现在说"我们落后了——加大力度。"下一步：又一个 30V 命令，又被限幅。控制器永远困惑，永远在命令它永远拿不到的电压。这就是朴素饱和产生振荡、超调，甚至失稳的原因——不是因为电机应付不了 12V，而是因为*控制器是为 30V 存在的世界设计的。*

**第二层——约束最优解的形状完全不同。** 当你把约束内建进 QP 求解时，优化器知道自己只有 12V。它不是把无约束解削顶——它找到的是一个*完全不同的控制序列*，在整个时域内都保持在 12V 以内，同时最小化跟踪代价。这通常意味着：

- 比无约束解**更早**开始减速，因为有限电压刹不住那么猛
- **更早减小电压**以避免冲过目标（冲过之后无力快速纠正）
- 朴素限幅会拍死到 12V 的某些步，优化器选择**根本不饱和**——它知道现在用 11.9V 比撞到 12V 后冲过头、再需要 −12V 拉回来更好

核心区别：朴素饱和说的是"你尽力去做，搞砸了我来擦屁股。"约束优化说的是"给定这些墙壁，找到一条从不无谓触碰它们的最佳路径。"两者只有在约束永不被触发的平凡情形下才等价。

**仿真器证明了这一点。** 在 `servo_qp_mpc.html` 中，LQR 曲线（橙色）和 QP-MPC 曲线（青色）跑在同一台电机上，用相同的代价权重。当约束活跃时，QP-MPC 的电压曲线不只是 LQR 曲线的削顶版——结构上就不一样。观察大幅阶跃参考时的电压图：LQR 命令一个被削顶的脉冲，超调，然后振荡。QP-MPC 提前收敛，给出更干净、通常整定更快的响应，总能耗更少。

**更深层的原理：** 每个物理系统都有约束——电压、电流、转矩、温度、压力。一个不知道它们在的最优控制器，在回答错误的问题。QP-MPC 答的是对的那个：*给定我实际拥有的东西，我能做到的最好是什么？*

### 🔑 控制问题如何变成 QP —— 冷凝法详解

每个时间步求解的 QP 并非凭空出现。它来自**消去 MPC 优化中的动力学约束**，这个过程称为*冷凝（condensing）*。以下是完整推导，与 `servo_qp_mpc.py` 中的代码一一对应。

#### 1. 原始的 MPC 问题

每个时间步，给定当前状态误差 `x₀_err = x − x_ref`，我们希望求解：

```
min    Σᵢ₌₀ᴺ⁻¹ (xᵢᵀ Q xᵢ + uᵢᵀ R uᵢ)   +   x_Nᵀ P x_N
u₀…uₙ₋₁

s.t.   xᵢ₊₁ = A_d xᵢ + B_d uᵢ        (离散电机动力学)
      −V_max ≤ uᵢ ≤ V_max            (电压限制)
```

终端代价 `x_Nᵀ P x_N`（其中 `P` 来自离散代数 Riccati 方程）保证稳定性——它近似预测窗口之外的无穷时域代价。

这个问题既有动力学等式约束，也有盒子不等式约束。通用 QP 求解器无法高效处理任意等式约束——我们需要消去动力学。

#### 2. 冷凝 —— 将状态轨迹写成纯 `x₀` 和 `U` 的函数

沿预测时域展开线性动力学。以 `N = 3` 为例：

```
x₁ = A_d x₀ + B_d u₀
x₂ = A_d² x₀ + A_d B_d u₀ + B_d u₁
x₃ = A_d³ x₀ + A_d² B_d u₀ + A_d B_d u₁ + B_d u₂
```

将所有 N 个未来状态堆叠成一个大向量 `X = [x₁; x₂; …; x_N]`：

```
X = A_aug · x₀_err  +  B_aug · U
```

其中：
- `A_aug` 是 `A_d, A_d², …, A_dᴺ` 的纵向堆叠——每一步对初始状态的*自由响应*
- `B_aug` 是一个**下三角块 Toeplitz 矩阵**——元素 `(row, col)` 为 `A_d^(row−col) B_d`，描述第 `col` 步的输入如何通过动力学传播到第 `row` 步的状态

对应 `servo_qp_mpc.py` 中的代码：

```python
A_aug = np.vstack([np.linalg.matrix_power(Ad, k+1) for k in range(N)])

B_aug = np.zeros((N*n, N*m))
for row in range(N):
    for col in range(row + 1):
        B_aug[row*n:(row+1)*n, col] = (
            np.linalg.matrix_power(Ad, row-col) @ Bd).flatten()
```

#### 3. 代入代价函数 —— 只剩标准 QP

原始代价为 `J = Xᵀ Q̄ X + Uᵀ R̄ U`，其中 `Q̄ = block-diag(Q,…,Q,P)` 将 Riccati 矩阵 `P` 放在最后一个块，`R̄ = block-diag(R,…,R)`。

**代入** `X = A_aug x₀_err + B_aug U` 到 `J` 中并展开：

```
J = x₀ᵀ A_augᵀ Q̄ A_aug x₀          ← 常数项（不依赖 U，可以去掉）
  + 2 x₀ᵀ A_augᵀ Q̄ B_aug U        ← U 的线性项
  + Uᵀ (B_augᵀ Q̄ B_aug + R̄) U     ← U 的二次项
```

定义：

```
H = B_augᵀ Q̄ B_aug + R̄              (二次代价矩阵——惩罚控制能量)
F = A_augᵀ Q̄ B_aug                   (将初始状态映射为线性代价项)
```

代码：

```python
Qbar = np.kron(np.eye(N), Q_lqr)
Qbar[-n:, -n:] = P_lqr               # 终端代价

H = B_aug.T @ Qbar @ B_aug + Rbar    # Uᵀ H U / 2
F = A_aug.T @ Qbar @ B_aug           # (Fᵀ x₀_err)ᵀ U
```

原始 MPC 问题现在等价于**标准盒子约束 QP**：

```
min    ½ Uᵀ H U  +  (Fᵀ x₀_err)ᵀ U
 U

s.t.   −V_max ≤ uₖ ≤ V_max      (仅盒子约束——每个控制步一个)
```

所有动力学约束都已被吸收进 `H` 和 `F`。QP 求解器只看到 N 个标量变量及其简单的上下界——这正是它能在 1 ms 控制回路上跑得动的原因。

#### 4. 构建一次 QP，反复求解

QP 的**结构**（`H`、`F`、盒子约束）在仿真循环之前一次性构建——`H` 和 `F` 只依赖电机模型和代价权重，这些在运行中不变。每个时间步变化的是当前状态误差 `x₀_err`，它通过线性项 `(Fᵀ x₀_err)ᵀ U` 进入问题。

这通过 cvxpy 中的**参数（Parameter）**来实现：

```python
x0_param = cp.Parameter(n)                 # 占位符：当前状态误差
U = cp.Variable(N)                         # 我们要优化的 N 个控制输入

prob = cp.Problem(
    cp.Minimize(0.5 * cp.quad_form(U, H) + (F.T @ x0_param).T @ U),
    [U >= -V_max, U <= V_max])
```

`Problem` 对象只构造一次。在每个 1 ms 时间步：

```python
x0_param.value = x0_err                    # 注入当前状态误差
prob.solve(solver=cp.OSQP, warm_start=True, polish=False,
           max_iter=400, eps_abs=1e-4, eps_rel=1e-4)
u = float(U.value[0])                      # 只施加第一个控制量（滚动时域）
```

关键求解器选择及其对实时控制回路的意义：

- **`solver=cp.OSQP`** — OSQP 是基于算子分裂（ADMM）的 QP 求解器。它对 MPC 很快，因为能利用冷凝问题的稀疏性，且热启动效果好。
- **`warm_start=True`** — 用上一个时间步的解初始化求解器。由于状态仅变化了一个 1 ms 步长，上一轮的优化序列非常接近正确答案，通常能减少 80% 以上的迭代次数。
- **`polish=False`** — 跳过最终的精细化求解步骤。在 1 ms 控制回路中，速度比高精度最优性更重要。
- **`max_iter=400, eps_abs/eps_rel=1e-4`** — 限制迭代次数并设定收敛容差。这些参数针对 1 ms 周期调优：如果求解器在 400 次迭代内尚未收敛，当前最佳解已足够好。
- 只有**第一个元素** `U.value[0]` 被施加到电机。序列的其余部分被丢弃，整个优化在下一个采样时间重新进行（滚动时域原理）。

#### 总结图

```
每个 1 ms 步：
  ┌─────────────┐     ┌─────────────────────┐      ┌──────────────┐
  │ 测量状态    │ ──>  │ 冷凝为 QP           │  ──> │ OSQP 求解    │
  │ 误差 x₀     │      │ H、F 已预计算       │      │ min ½UᵀHU +  │
  │             │     │ x₀ -> 线性项        │      │ (Fᵀx₀)ᵀU     │
  └─────────────┘     └─────────────────────┘      │ s.t. |u|≤V │
                                                   └──────┬───────┘
                                                          │
                                         只施加 u₀  ───────┘
```

与朴素饱和的关键区别：QP 在规划轨迹时*知道约束的存在*——`H` 和 `F` 编码了完整的动力学，因此求解器可以提前收敛电压以避免超调。朴素饱和却假设无限电压存在（从无约束 Riccati 方程计算增益），事后削足适履——这就是它会超调和振荡的根本原因。

#### 5. 求解器速度对比：OSQP vs DAQP

冷凝后的 QP 只有 `N` 个变量和简单的盒子约束——但要在 1 ms 内求解它，需要一个快速的求解器。`servo_qp_mpc.py` 现已添加两个求解器在相同问题上的基准对比（12 变量, 1500 次求解, 1 ms 步长）：

| 求解器 | 平均 [µs] | 中位数 [µs] | 最大 [µs] | 总计 [ms] |
|--------|-----------|-------------|----------|------------|
| OSQP (via cvxpy) | 409 | 391 | 7635 | 614 |
| DAQP (direct C API) | 3.9 | 3.5 | 69 | 5.8 |

**DAQP 每次求解快约 105 倍。** 差距来自两个因素：

- **cvxpy 建模层开销** — 即使 `Parameter` 已经设置好，cvxpy 每次调用时仍然将符号化的 `Problem` 编译为求解器标准形式。在小规模问题上，这个开销远远超过 OSQP 的实际求解时间。
- **DAQP 的对偶主动集方法** — DAQP 直接通过 C API 利用问题的小规模（12 变量）和简单结构（仅有盒子约束）。在小型稠密 QP 上，主动集方法通常比 OSQP 的算子分裂法（ADMM）收敛更少的迭代，因此快得多。

对于这个演示问题，两种求解器都能轻松满足 1 ms 的时限。但在以下场景中，速度差异至关重要：

- **更大的预测时域** — 冷凝后的 `H` 矩阵按 N×N 增长，QP 会变得更稠密、更难解
- **更快的控制回路** — 100 kHz 电机驱动、电力电子设备，控制周期仅 10–100 µs
- **嵌入式部署** — cvxpy 需要 Python 运行环境；C 原生求解器如 DAQP 可以编译到微控制器和 DSP 上

`servo_qp_mpc.py` 在每次运行结束后会打印完整的基准对比表。

## 交互式零点效应探索器

`zero_effect_explorer.html` 是一个约 1180 行的单文件 HTML，零依赖：

- **实时阶跃响应**——通过状态空间（能控标准型）+ RK4 仿真计算
- **实时零极点图**——展示参数调整时极点和零点的移动
- **滑块**——阻尼比 ζ、自然频率 ωₙ，以及最多 3 个零点（每个可跨越 LHP ↔ RHP）
- **7 个预设**——覆盖关键场景：远/近 LHP 零点、远/近 RHP 零点、双零点、混合
- **动态解读**——底部面板根据当前配置解释你看到的现象
- **合成曲线**——当激活 2 个以上零点时，白色虚线显示它们的复合效果
