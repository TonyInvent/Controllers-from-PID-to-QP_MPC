## 你将学到什么

**PID 探索器**——控制器增益如何重塑闭环动态：
- Kp 提高有效 ωₙ（响应更快），但降低有效 ζ（超调更大）
- Kd 增加阻尼——将主导极点向左拉，提升 ζ
- Ki 通过在 s=0 处引入积分器极点来消除稳态误差
- PD 让你可以独立调节 ζ_eff 和 ωₙ_eff；PI 消除静差但会拖长调节时间
- 有效 ζ 和 ωₙ 由闭环主导极点对计算得出——底层使用卡尔丹诺三次方程公式

**零点效应探索器**——零点如何改变二阶系统的阶跃响应：

**左半平面（LHP）零点**——速度与超调的权衡：
- 远离原点的零点几乎不可见
- 零点越靠近原点，微分"冲击"越大，超调增加但上升时间缩短
- 这正是 PID 微分作用在做的事——你在放置一个零点，而*放在哪里*至关重要

**右半平面（RHP）零点**——硬性的物理限制：
- 系统初始会朝**错误的方向**运动，之后才恢复
- 越靠近原点 = 反调越深、越持久
- 任何控制器都无法消除 RHP 零点；它施加了大约 |z|/2 的带宽限制
- 现实案例：锅炉汽包水位——加入冷水后水位先下降再上升

**多个零点**——效果叠加：
- 每增加一个零点就增加一个微分项
- 两个 LHP 零点比单个零点的超调更大

### 🔑 一个核心洞见：积分器极点会移动

当你滑动 Ki 滑块时，会立刻看到以下现象——这**不是**计算错误：

PID 控制器的积分项（Ki/s）在 **s = 0** 处引入一个极点。在**开环**中，这个极点就停在原点。但当反馈回路闭合后，它**移动了**。以"PID Tuned"预设为例（ζ=0.5, ωₙ=2, Kp=3, Kd=0.8）：

| Ki | 实极点（原本在 s=0） | 共轭复极点对 | 现象 |
|----|----------------------|-------------|------|
| 0（PD） | 抵消 | −2.6 ± j3.0 | 无积分器——稳态误差残留 |
| 0.4 | **−0.103** | −2.55 ± j3.0 | 实极点靠近原点 → 拖出长长的调节尾巴 |
| 2 | −0.48 | −2.36 ± j3.2 | 积分器加速，共轭复极点对向右漂移 |
| 10 | −3.77 | −0.71 ± j3.2 | 共轭复极点对逼近虚轴——Ki 太大了！ |

**这是 PID 控制的基本特性**：积分作用将稳态误差拉向零，但随着 Ki 增大，实极点向左移动（积分器加快），而共轭复极点对向虚轴靠近。Ki 调得太大，复极点将越过虚轴进入右半平面——系统失稳。

从根轨迹的角度看：积分器极点从 s=0 出发沿负实轴移动，而两个对象极点先彼此靠拢，然后随 Ki 增大分离。PID 探索器实时展示这一切——滑动 Ki 时请紧盯零极点图。

## 伺服电机 PID 演示

`servo_motor_pid.html` 将 PID 落地到真实的物理系统——**有刷直流电机**伺服位置控制：

- **三阶电机模型**——物理参数：电枢电阻 R（Ω）、电感 L（H）、转矩常数 Kt（N·m/A）、转子惯量 J（kg·m²）、粘性摩擦 B（N·m·s/rad）
- **真实约束**——电压饱和（±Vmax）配合抗积分饱和、干扰转矩注入

- **实时零极点图**——开环电机极点（s=0、机械极点、电气极点）、闭环极点（四阶，通过 Ferrari 四次方程求解）、控制器零点
- **刚性 ODE 处理**——当电气时间常数过小而无法用显式 RK4 积分时，自动切换为降阶模型
- **7 个预设**——无控制、纯 P、PD、PID 出厂整定、高惯量、电压饱和、干扰负载
- **侧边栏 SVG 电机示意图**——转子、线圈、转轴、接线端
- **性能读数**——带宽、ζ_eff、超调百分比、稳态误差、峰值电压、调节时间

这个演示回答了：当你把 Kp 调得很大时，真实电机会发生什么？为什么微分增益（Kd）能防止超调？积分器（Ki）如何消除恒定转矩干扰？

### 为什么闭环极点数量随 Ki 变化

一个微妙但重要的细节：切换到 PD 预设（Ki=0），你会看到 **3** 个闭环极点。切换到 PID（Ki>0），你会看到 **4** 个。这不是 bug——是数学的直接结果。

电机的开环传递函数是三阶的：

$$G(s) = \frac{K_t}{LJ s^3 + (LB+RJ) s^2 + (K_t^2+RB)s}$$

分母中的 $s$ 因子是机械积分器（角度 = ∫ 角速度 · dt）。PID 控制器 $C(s) = K_p + K_d s + K_i/s$ 通过 $K_i/s$ 项再增加一个积分。

- **Ki > 0（完整 PID）：** 为了消除 $K_i/s$ 项，必须两边同乘 $s$，得到 **四阶四次方程**：$LJ s^4 + (LB+RJ)s^3 + (K_t^2+RB+K_t K_d)s^2 + K_t K_p s + K_t K_i = 0$。四个极点。
- **Ki = 0（PD）：** 没有 $K_i/s$ 需要消除。特征方程直接是 **三阶三次方程**：$LJ s^3 + (LB+RJ)s^2 + (K_t^2+RB+K_t K_d)s + K_t K_p = 0$。电机在 $s=0$ 处的开环极点在 PD 反馈下**移动了**——$K_p$ 和 $K_d$ 将其沿负实轴向左推动。三个极点。

在 $K_i=0$ 时额外乘以 $s$ 会在原点引入一个虚假根——这是数学操作带来的伪影，而非物理极点。代码正确处理：PD 时使用三次方程求解器，完整 PID 时使用四次方程求解器。

## LQR 探索器

`lqr_explorer.html` 在同一有刷直流电机上展示了 **LQR（线性二次型调节器）** 设计——PID 的现代状态空间对应方法：

- **LQI（LQR + 积分作用）**——将三阶电机模型扩展一个误差积分状态，实现零稳态跟踪误差
- **调节权重，而非增益**——不再试探性地调整 Kp/Kd/Ki，而是指定**代价权重** Q（状态惩罚）和 R（控制量惩罚）；最优状态反馈增益 K 自动计算得出
- **CARE 求解器**——通过 Hamiltonian 方法求解连续时间代数 Riccati 方程，使用 `numeric.eig()` 对 8×8 Hamiltonian 矩阵进行特征值分解
- **实时阶跃响应**——双 Y 轴图：左轴 θ [rad]、右轴电压 V [V]；接近 ±Vmax 时显示饱和填充区域
- **实时零极点图**——4 个 LQR 闭环极点（青色 ×）对比 3 个开环电机极点（灰色 ×）；高亮稳定性边界
- **对数映射的代价权重**——滑块对应 q_θ（位置误差）、q_ω（速度）、q_i（电流）、q_∫（积分误差）和 R（控制量惩罚）
- **PID 对比开关**——叠加 PID 阶跃响应（金色虚线），直观对比经典整定与最优 LQR
- **增益读数**——K = [k_θ, k_ω, k_i, k_∫]，颜色编码对应 PID 概念：k_θ≈Kp、k_ω≈Kd、k_i（电流反馈）、k_∫≈Ki
- **6 个预设**——均衡 LQR、低成本控制、高成本控制、位置优先、重阻尼、快速积分器
- **抗积分饱和**——电压饱和时冻结积分器状态，防止积分器卷绕
- **自适应时间窗口**——仿真时长根据闭环调节时间自动缩放（4/(ζ·ωₙ)）

### LQR vs PID：核心区别

| 方面 | PID | LQR |
|------|-----|-----|
| 整定方式 | 试探——Kp、Kd、Ki | 系统化——Q、R 权重 |
| 增益数量 | 3 个标量增益 | 4 个增益：k_θ、k_ω、k_i、k_∫ |
| 电流反馈 | 无（隐含） | 显式 k_i 反馈 |
| 设计方法 | 边试边看 | 一步最优 |
| 理论保证 | 无（可能失稳） | 保证稳定裕度（≥60° PM, ≥6 dB GM） |
| 工作量 | 直接调节响应形状 | 惩罚状态 + 控制量 → 最优折中 |

### LQI 代价函数

$$J = \int_0^\infty \left( q_\theta \theta^2 + q_\omega \omega^2 + q_i i^2 + q_{\int} (\smallint e)^2 + R u^2 \right) dt$$

- **q_θ ↑**——位置响应更快，超调增大
- **q_ω ↑**——阻尼增加，超调减小
- **q_i ↑**——电流消耗更小
- **q_∫ ↑**——稳态误差恢复更快
- **R ↑**——控制量更小、更保守（惩罚电压）

CARE 求解器得出最小化 J 的最优增益矩阵 **K**。闭环特征值 (A − BK) 即为 LQR 最优极点位置——无需手动配置极点。

## 交互式零点效应探索器

`zero_effect_explorer.html` 是一个约 1180 行的单文件 HTML，零依赖：

- **实时阶跃响应**——通过状态空间（能控标准型）+ RK4 仿真计算
- **实时零极点图**——展示参数调整时极点和零点的移动
- **滑块**——阻尼比 ζ、自然频率 ωₙ，以及最多 3 个零点（每个可跨越 LHP ↔ RHP）
- **7 个预设**——覆盖关键场景：远/近 LHP 零点、远/近 RHP 零点、双零点、混合
- **动态解读**——底部面板根据当前配置解释你看到的现象
- **合成曲线**——当激活 2 个以上零点时，白色虚线显示它们的复合效果
